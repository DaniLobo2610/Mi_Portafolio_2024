## Contexto del proyecto
Este proyecto utiliza un sistema Grpc para consultar informacion de luchadores de Mortal Kombat.
utilizando los cuatro tipos de servicios:
- Unary: Un servicio que recibe un solo mensaje y devuelve un solo mensaje.
- Client Streaming: Un servicio que recibe un mensaje y devuelve una secuencia de mensajes.
- Server Streaming: Un servicio que recibe una secuencia de mensajes y devuelve un mensaje.
- Bidirectional Streaming: Un servicio que recibe y devuelve secuencias de mensajes.

El sistema oficial será implementado en azure, pero igualmente funciona de manera local, donde la informacion de los fighters será
consultada mediante tres variables:
- Name
- Type
- Skills
